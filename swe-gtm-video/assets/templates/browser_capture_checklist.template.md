# Browser Capture Checklist — {{PROJECT_NAME}}

## Pre-capture

- [ ] Correct app build/version
- [ ] Correct feature flags
- [ ] Seeded demo data loaded
- [ ] No PII, secrets, customer names, internal URLs, or private docs visible
- [ ] Browser zoom set to 100% unless intentionally different
- [ ] Viewport set to target resolution/aspect ratio
- [ ] Notifications, browser extensions, debug panels, and unrelated tabs hidden
- [ ] Cursor settings decided: visible, hidden, or stylized
- [ ] Brand assets and UI theme correct

## During capture

- [ ] Move deliberately
- [ ] Pause after meaningful UI state changes
- [ ] Avoid accidental hover/clicks
- [ ] Record multiple takes for key moments
- [ ] Capture still frames for generated video reference

## Post-capture

- [ ] Rename files with capture IDs
- [ ] Log captures in shot list and asset registry
- [ ] Review for UI accuracy and privacy
- [ ] Retake anything blurry, cluttered, or misleading

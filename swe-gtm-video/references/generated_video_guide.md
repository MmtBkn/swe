# Generated Video Guide

## Use generated video for

- Emotional context
- Abstract metaphors
- Transitions
- Background plates
- Camera moves around approved screenshots
- Non-product story beats

## Do not use generated video for

- Fake product functionality
- Fake customer proof
- Recreated logos
- Screens where exact UI accuracy matters and the model cannot preserve it
- Claims or scenarios not approved by product/legal/marketing

## Grounded software-screen prompt pattern

```text
Use the provided screenshot as an immutable software interface layer. Preserve every pixel of the UI, including text, layout, color, icons, spacing, and logo marks. Add only subtle camera movement and depth outside the interface. The screen should remain readable and accurate. No new UI elements, no changed labels, no invented data, no distorted logos.
```

## Cinematic B-roll prompt pattern

```text
Create a restrained cinematic visual metaphor for {{persona pain or transformation}}. The image should feel modern, human, and premium, avoiding generic sci-fi and avoiding any visible brand logos. Motion should be subtle and purposeful, supporting a software GTM video.
```

## Acceptance criteria

- The generated asset supports a specific timestamp and story beat.
- It does not replace real product proof.
- It passes brand and truth review.
- It can be edited cleanly with captions, VO, and UI captures.

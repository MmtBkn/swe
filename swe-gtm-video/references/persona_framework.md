# Persona Framework for SWE GTM Videos

## Persona dimensions

For each persona, capture:

- Role and environment
- Stakes: what happens if the workflow fails?
- Pressure: time, risk, quality, cost, compliance, reputation, career
- Current workaround
- Emotional state before the product appears
- Desired identity after success
- Misconception or objection
- Proof required to trust the product
- Language the persona actually uses

## Persona-to-story mapping

| Persona signal | Story decision |
|---|---|
| Time pressure | Open with compression, overload, or clock tension |
| Risk/compliance pressure | Use restraint, evidence, auditability, and clear proof |
| Creativity/innovation desire | Use a more exploratory visual language |
| Technical skepticism | Show real UI, real paths, and precise claims |
| Executive buyer | Make outcome and strategic leverage visible |
| Practitioner/user | Make daily workflow relief visible |

## Misconception mapping

Turn misconceptions into scenes:

- "AI will be another tool I need to babysit" → show delegated outcome with clear control.
- "This will not understand our workflow" → show grounded context and accurate handoff.
- "This is just a chatbot" → show orchestration across systems and concrete action.
- "Adoption will be risky" → show guardrails, review, audit trail, and reversibility.

## Output expectation

The persona section should make the script easier to write. If it does not create obvious scenes, it is too abstract.

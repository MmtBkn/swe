# Story Framework for Feature-to-Film Conversion

## Core structure

Use this default spine unless the launch context suggests a different structure:

1. Hook — a felt problem or provocative contrast.
2. Before — the current workflow, not as complaint but as lived reality.
3. Inciting shift — the feature appears as a change in agency.
4. Proof — real UI or credible output demonstrates the shift.
5. Expansion — the viewer understands the bigger implication.
6. CTA — a simple next move.

## Line-level rules

- Make the first line human, not product-first.
- Avoid "Introducing..." unless the visual already carries tension.
- Prefer verbs over nouns.
- Use one memorable line; do not make every line sound like a tagline.
- Use fewer words than feels comfortable.
- Let the product proof carry explanation.

## Strong line patterns

- "The work did not disappear. The waiting did."
- "Not another place to ask. A place where work moves."
- "The handoff is no longer where momentum dies."
- "Your team keeps control. The agent carries the load."

Use examples only as patterns. Create original copy for the product.

## Cliche blacklist

Avoid:

- "Revolutionize your workflow"
- "Unlock productivity"
- "Game-changing"
- "Seamless experience"
- "AI-powered magic"
- "Do more with less"
- "The future is here"

Replace with concrete stakes and proof.
